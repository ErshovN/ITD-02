# ITD-02
### Лабораторная работа №2: Реализация проверки простых правил DRC.

### Цель работы:
Реализовть алгоритм проверки базовых правил проектирования (Design Rule Check, DRC) для простой топологии схемы на языке Python. Освоить алгоритмы парсинга, проверки геометрических ограничений и формирования отчёта о нарушениях.

### Задача:
1. Реализовать парсинг JSON из 1-й лаб. работы.
2. Реализовать алгоритм проверки базовых правил проектирования в зависимости от варианта

| Тип алгоритма                              | Вариант |
|--------------------------------------------|---------|
| **Булевы операции и геометрическое расширение** | 1, 4, 7, 10, 13, 16, 19, 22, 25, 28|
| **Алгоритм сканирующей прямой**            | **2, 5, 8, 11, 14, 17, 20, 23, 26, 29** |
| **Алгоритм сеточной (растрированной) проверки** | 3, 6, 9, 12, 15, 18, 21, 24, 27, 30 |

3. Сформировать отчет (название файла группа_ФИО_алгоритм.json).

   В результате работы алгоритма должен формироваться json файл с нарушениями
   
### Пример сформированного файла
```json
{
  "detailed_predictions": [
    {
      "source_element": 0,
      "target_element": 2,
      "violation_probability": 1,
      "status": "CRITICAL"
    },
   // ... остальные нарушения
}
```
 ### Описание полей
| Поле | Значение | Описание |
|------|----------|----------|
| **`source_element`** | `0, 5, 12...` | ID полигона-нарушителя №1 |
| **`target_element`** | `1, 8, 15...` | ID полигона-нарушителя №2 |
| **`violation_probability`** | **`1.0`** | **Всегда 1 (определенное нарушение)** |
| **`status`** | **`CRITICAL`** | **Всегда CRITICAL** |
